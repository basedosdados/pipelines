# us_census_cbp — session context

County Business Patterns (CBP), U.S. Census Bureau. Onboarding to Data Basis.
**Read `ONBOARDING_PLAN.md` in this folder first** — it is the architecture source of truth.

## What this dataset is
Annual counts of business **establishments, employment (mid-March), and payroll**
by **geography × industry (NAICS) × legal form × establishment-size**. Public domain.
GCP id `us_census_cbp` · org `us_census` · backend slug `cbp`.

## Tables (WIDE — one row per source row; size bands are columns)
- `national` — year × naics × lfo (84 cols; from `cbpNNus`)
- `state` — year × id_state × naics × lfo (85 cols; from `cbpNNst`)
- `county` — year × id_state × id_county × naics (26 cols; no lfo, counts-only per band; from `cbpNNco`)
- `dicionario` — labels for lfo, naics_version, flags

Keys are **id-first** (`id_state`, `id_county`, `naics`) to match `br_bd_diretorios_us`.
The wide column grid is generated by `code/gen_arch_sheets.py` — edit that, not the sheets, if columns change.
(Design note: an earlier LONG-on-size_class design was reversed — grounded county row counts
were 1.10M/yr, so long → ~150–400M rows with ~90% null $; wide is ~38M total and canonical.)

## Non-obvious things that will bite you
1. **NAICS is multi-vintage.** CBP 1998–2023 spans NAICS 1997/2002/2007/2012/2017/2022.
   2017+ matches the `naics_2022` directory exactly; older years drift (2012: 2%,
   2008: 11.5%, earlier worse). We carry a `naics_version` column and enforce
   referential integrity **per vintage** in dbt, against per-vintage directories
   `br_bd_diretorios_us__naics_<v>` (these must be built first — see plan §Prerequisite).
2. **NAICS field has aggregation fill** in raw files: `------`(all-industry total,
   DROP), `11----`(sector), `113///`/`1131//`/`11311/`(slashes), `113110`(6-digit).
   Cleaning strips dash/slash fill to the native code. Sector split-codes 31/32/33,
   44/45, 48/49 stay native (directory stores ranges `31-33` etc.) → excluded from naics test.
3. **US/state ≠ county schema, and the schema shifts by era.** US & state files carry
   emp/payroll per size-band and per `lfo`; county files carry only establishment
   counts per band + totals. Also: 1998–2007 US/state have **no `lfo`**; noise flags
   (`emp_nf`, `eSnf`, …) exist only **2017+**; `empflag`/`fS` suppression flags exist
   **1998–2016**; the smallest band is `n<5` in modern files, `n1_4` in older ones
   (same band → column `establishments_1_4`). Clean **by header name**, not position;
   fill absent columns with null.
4. **Two disclosure eras.** 1998–2016 = suppression (`empflag` letter, `emp`=0 when
   withheld). 2017–2023 = noise infusion (`emp_nf`/`qp1_nf`/`ap_nf` = G/H/J). Keep both
   flag families; do not drop rows.
5. **Payroll is published in $1,000s** — multiply by 1000 at clean time; unit = USD.
6. Drop legacy `censtate`/`cencty` (redundant with FIPS).

## Conventions (see .claude/rules/)
- Types by arithmetic meaning: coded/flag columns → STRING + `covered_by_dictionary=yes`.
- Every INT64/FLOAT64 needs a `measurement_unit` (establishment, employee, USD).
- No `is_primary_key` on these tables (non-directory); logical key enforced in dbt only.
- Backend env: **staging** per standing directive (dev down 2026-07-09; re-check).
