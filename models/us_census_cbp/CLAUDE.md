## Non-obvious things that will bite you
1. **NAICS is multi-vintage, and CBP lags the revisions.** CBP 1998â€“2023 spans five
   vintages: 1997/2002/2007/2012/2017. There is **no 2022 vintage** â€” CBP still
   publishes NAICS 2017 through 2023, so 2022/2023 data are NAICS 2017, and the
   official `naics_2022` directory is never referenced. Against `naics_2022` the
   older years drift badly (2012: 2%, 2008: 11.5%, earlier worse), which is why we
   carry a `naics_version` column and enforce referential integrity **per vintage**
   in dbt against `br_bd_diretorios_us__naics_<v>` (build these first â€” see plan
   Â§Prerequisite). Measured: 0.000% unresolved for 2002/2007/2012/2017 and 0.094%
   for 1997 (only the `95`/`99` admin codes, which the test ignores), so every
   vintage test runs at **zero** tolerance.
2. **NAICS field has aggregation fill** in raw files: `------`(all-industry total,
   DROP), `11----`(sector), `113///`/`1131//`/`11311/`(slashes), `113110`(6-digit).
   Cleaning strips dash/slash fill to the native code. Sector split-codes 31/32/33,
   44/45, 48/49 stay native and need **no test exception**: the per-vintage dirs are
   built from CBP's own reference files, so they store `31`/`44`/`48` natively rather
   than the ranges (`31-33`) the official `naics_2022` dir uses. Verified: zero
   unmatched split-sector rows in any vintage.
3. **US/state â‰  county schema, and the schema shifts by era.** US & state files carry
   emp/payroll per size-band and per `lfo`; county files carry only establishment
   counts per band + totals. Also: 1998â€“2007 US/state have **no `lfo`**; noise flags
   (`emp_nf`, `eSnf`, â€¦) exist only **2017+**; `empflag`/`fS` suppression flags exist
   **1998â€“2016**; the smallest band is `n<5` in modern files, `n1_4` in older ones
   (same band â†’ column `establishments_1_4`). Clean **by header name**, not position;
   fill absent columns with null.
4. **Two disclosure eras.** 1998â€“2016 = suppression (`empflag` letter, `emp`=0 when
   withheld). 2017â€“2023 = noise infusion (`emp_nf`/`qp1_nf`/`ap_nf` = G/H/J). Keep both
   flag families; do not drop rows.
5. **Payroll is published in $1,000s** â€” multiply by 1000 at clean time; unit = USD.
6. Drop legacy `censtate`/`cencty` (redundant with FIPS).
